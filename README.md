# AI 一人公司生存地图

[在线预览：AI 一人公司生存地图](https://chenleiai.github.io/AI-One-Person-Company-Survival-Map/)

一个面向大众的交互式观察模型：用“全网 AI 一人公司生存地图 + 10 题生存体检 + 个性化报告”的形式，帮助职场人、老板、自由职业者、创作者、个体户和转型者判断自己在 AI 时代的位置。

## 这个项目想解决什么

很多人都知道 AI 会改变工作，但不知道自己的压力来自哪里。这个工具把问题落到个人身上：

- 哪些重复工作正在被 AI 重新定价？
- 你有没有能被 AI 放大的核心价值？
- 你当前 AI 生存成熟度是多少？
- 你最该补上的第一个 AI 员工是谁？
- 你每周有多少时间被低价值工作吞掉？

## 当前功能

- AI 一人公司生存地图
- 职场打工人、小微老板、专业服务、转型者、创作者、电商、本地服务、自由职业等样本
- 鼠标悬停快速预览，点击方块查看模式详情
- 10 题生存体检问卷
- 成熟度、AI重构压力、时间差、最该补的 AI 员工
- 全网样本库与案例库入口
- 朋友圈海报预览与图片保存

## 数据说明

当前版本是 MVP，内置的是“观察模型样例数据”，不是严谨统计研究，也不是收入承诺。

后续开源版本建议逐步补齐：

- 真实案例来源链接
- 案例采集表单
- AI 化程度评分规则
- 收入和时间字段的估算规则
- 每月更新日志

## 本地运行

### 方式一：双击启动

在 macOS 上直接双击项目里的：

```text
启动本地网站.command
```

脚本会自动打开：

```text
http://localhost:5173/
```

注意：关闭脚本打开的终端窗口后，本地网站也会停止。

### 方式二：命令行启动

```bash
npm install
npm run dev -- --host 0.0.0.0
```

构建：

```bash
npm run build
```

## 静态部署

这个项目是纯前端项目，不需要后端服务，也不需要配置 API Key。运行 `npm run build` 后会生成 `dist/` 目录，可以直接部署到 GitHub Pages、Vercel、Netlify 或任意静态网站托管服务。

当前 Vite 配置使用相对路径，适合部署在 GitHub Pages 的项目子路径下。

如果部署到 GitHub Pages，仓库已经内置自动部署流程。把代码推送到 `main` 分支后，在 GitHub 仓库的 Pages 设置里选择 `GitHub Actions` 作为发布来源即可。

## 开源建议

建议先以“观察项目”发布，不要包装成严肃行业报告。最重要的是把案例库做实，让用户相信这不是自嗨，而是持续研究全网 AI 一人公司后的沉淀。
